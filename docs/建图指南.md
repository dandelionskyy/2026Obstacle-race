# FAST-LIO2 建图与保存指南

## 概述

建图阶段只需 FAST-LIO2 + LiDAR 驱动，不需要启动 Nav2、状态机等完整比赛栈。

当前使用 **Livox MID-360** 激光雷达（倒装，roll=180°），配置文件为 `mid360.yaml`。

---

## 1. 启动 LiDAR 驱动

```bash
source ~/2026Obstacle-race/install/setup.bash
ros2 launch livox_ros_driver2 msg_MID360_launch.py
```

> 如果驱动未安装，参考 [Livox ROS Driver 2](https://github.com/Livox-SDK/livox_ros_driver2)。

---

## 2. 启动 FAST-LIO2 建图

**另开一个终端：**

```bash
source ~/2026Obstacle-race/install/setup.bash

# 有显示器 → 带 RViz 可视化
ros2 launch fast_lio mapping.launch.py rviz:=true

# 无显示器/SSH → 纯后台建图
ros2 launch fast_lio mapping.launch.py rviz:=false
```

启动后：
- 输出话题 `/cloud_registered` — 实时点云（odom 坐标系）
- 输出话题 `/Laser_map` — 累积地图点云（每 1 秒发布一次）
- 输出话题 `/Odometry` — 里程计位姿
- 输出话题 `/path` — 运动轨迹

然后**推着/遥控机器人**在需要建图的区域走一圈。RViz 中可以实时看到点云地图的累积。

---

## 3. 保存 PCD 地图

走完一圈后，在**第三个终端**调用保存服务：

```bash
ros2 service call /map_save std_srvs/srv/Trigger
```

PCD 文件会保存为 `scan.pcd`，位于**启动 FAST-LIO2 节点的终端的当前工作目录**下（由 `mid360.yaml` 中 `map_file_path: "./scan.pcd"` 决定）。

如果想保存到固定路径，可以修改配置文件中的路径，或者保存后手动拷贝。

---

## 4. 拷贝到 maps 目录

```bash
cp scan.pcd ~/fast-lio-nav-obstacle/maps/
```

---

## 5. （可选）转换为 Nav2 2D 代价地图

建好的 3D 点云地图需要转成 2D 栅格地图才能给 Nav2 用。

### 5.1 修改 pcd2pgm 配置

编辑 `src/mapper/pcd2pgm/config/pcd.yaml`，确保路径正确：

```yaml
/pcd2pgm:
  ros__parameters:
    file_directory: /home/dandelion/fast-lio-nav-obstacle/maps/
    file_name: scan              # 对应 scan.pcd
    map_resolution: 0.05         # 5cm 分辨率
    thre_z_max: 0.35             # Z 轴上限 (高于地面的障碍物)
    thre_z_min: -0.04            # Z 轴下限
    thres_point_count: 10
```

### 5.2 启动转换节点

```bash
source ~/2026Obstacle-race/install/setup.bash
ros2 launch pcd2pgm pcd2pgm.launch.py
```

节点会在 `/map` 话题上发布 2D 占据栅格地图。

### 5.3 保存为 PGM+YAML

```bash
ros2 run nav2_map_server map_saver_cli -f ~/fast-lio-nav-obstacle/maps/field_map
```

生成 `field_map.pgm` 和 `field_map.yaml`，Nav2 启动时加载这个 YAML 即可。

## 6. 关键配置回顾

配置文件：`src/lio/FAST_LIO/config/mid360.yaml`

| 参数 | 值 | 说明 |
|------|-----|------|
| `pcd_save.pcd_save_en` | `true` | 启用 PCD 保存服务 |
| `pcd_save.interval` | `-1` | -1 表示保存所有帧 |
| `map_file_path` | `"./scan.pcd"` | 保存路径（相对路径） |
| `publish.map_en` | `true` | 发布 `/Laser_map` 累积地图 |
| `publish.scan_publish_en` | `true` | 发布点云输出 |
| `filter_size_map` | `0.2` | 地图降采样分辨率 |
| `mapping.extrinsic_est_en` | `false` | 外参在线估计（建图建议关） |

---

## 7. RViz 可视化

建图时 RViz 使用 `fastlio.rviz` 或 `robocon_bringup/config/rviz_config.rviz`：

```bash
ros2 launch fast_lio mapping.launch.py rviz:=true rviz_cfg:=<path_to_rviz_config>
```

RViz 中可以看到：
- `/cloud_registered` — 当前帧点云
- `/Laser_map` — 累积的地图点云
- `/path` — 轨迹线
- TF 树：`odom → base_link → livox_frame`

## 8. 完整流程速查

```bash
# 终端 1: LiDAR
ros2 launch livox_ros_driver2 msg_MID360_launch.py

# 终端 2: 建图
ros2 launch fast_lio mapping.launch.py rviz:=true

# ... 推着机器人走一圈 ...

# 终端 3: 保存
ros2 service call /map_save std_srvs/srv/Trigger
cp scan.pcd ~/fast-lio-nav-obstacle/maps/

# （可选）生成 2D 地图
ros2 launch pcd2pgm pcd2pgm.launch.py
ros2 run nav2_map_server map_saver_cli -f ~/fast-lio-nav-obstacle/maps/field_map
```
