## 1. 系统概述

### 1.1 比赛背景

2026 年 Robocon 仿生足式机器人挑战赛 — **障碍赛**：

| 项目 | 参数 |
|------|------|
| 场地尺寸 | 14m × 6m |
| 时间限制 | 210 秒 |
| 控制模式 | 全自主（150分/障碍）或遥控（100分/障碍） |
| 障碍物数量 | 8 个 |

### 1.3 系统设计目标

- ✅ **实时性**: 所有算法节点 C++ 实现, 100Hz TF 发布
- ✅ **鲁棒性**: 多传感器融合 (LiDAR+IMU+相机), AprilTag 漂移修正

- ✅ **可调试**: SSH 终端监控 + 航点测试模式
- ✅ **模块化**: 11 个独立功能包, 松耦合架构

## 4. 坐标系与 TF 树

### 4.1 TF 变换链

```
map ──→ odom ──→ base_link ──→ livox_frame   [倒装: roll=π, 180°]
                             ├─→ camera_link  [前向安装]
                             └─→ imu_link     [与livox同刚体]
```

### 4.2 各帧含义

| 帧名 | 含义 | 发布者 | 频率 |
|------|------|--------|------|
| `map` | 全局世界坐标系 | localization_manager | 100Hz |
| `odom` | 里程计坐标系 (漂移) | FAST-LIO2 | 100Hz |
| `base_link` | 机器人本体坐标系 | FAST-LIO2 | 100Hz |
| `livox_frame` | LiDAR 坐标系 (倒装) | static_transform_publisher | 静态 |
| `camera_link` | D455 相机坐标系 | static_transform_publisher | 静态 |
| `imu_link` | IMU 坐标系 | static_transform_publisher | 静态 |

### 4.3 map→odom 变换逻辑

```
启动时:
  map→odom = Identity (map与odom原点重合)
  └── 无先验地图也能运行, 机器人起点 = map原点

运行时 AprilTag 修正:
  if (检测到已知世界坐标的Tag):
    1. 反算 T_map_base = T_map_tag × inv(T_camera_tag) × T_camera_base
    2. 异常值检测: |平移跳变|<0.5m 且 |旋转跳变|<15°
    3. EMA 滤波: new = α × observed + (1-α) × old  (α=0.3)
    4. 发布修正到 /tag_pose_correction
    5. localization_manager 更新 map→odom

后续 (待实现):
  ICP 初始对齐: 将当前点云与预建 PCD 地图匹配, 一次性设置 map→odom
```

### 4.4 静态 TF 参数

```
base_link → livox_frame:
  translation: (0.15, 0.0, 0.35)    # 头部位置
  rotation: (π, 0, 0)               # 绕x轴180° (倒装朝下)

base_link → camera_link:
  translation: (0.25, 0.0, 0.30)    # 前向, 居中
  rotation: (0, 0, 0)               # 无旋转

base_link → imu_link:
  translation: (0.15, 0.0, 0.35)    # 与livox_frame相同
  rotation: (π, 0, 0)               # 与livox_frame相同
```

### 5.2 FAST_LIO — 激光-惯性里程计

**来源**: 香港大学 MARSLab, [FAST-LIO2](https://github.com/hku-mars/FAST_LIO)

**核心算法**:
1. 迭代误差状态卡尔曼滤波 (IESKF)
2. 直接点-地图配准 (无需特征提取)
3. ikd-Tree 增量式动态 KD 树

**关键技术参数** (mid360.yaml):

```yaml
preprocess:
  lidar_type: 1          # Livox系列 (MID360=4, 实际用1)
  scan_line: 4
  blind: 0.5             # 盲区距离 (m)
  scan_rate: 10          # 扫描频率 (Hz)
  point_filter_num: 3    # 每隔3个点取1个

mapping:
  acc_cov: 1.2           # 加速度计协方差
  gyr_cov: 1.2           # 陀螺仪协方差
  extrinsic_est_en: true # 在线外参估计
  det_range: 100.0       # 检测范围 (m)
  fov_degree: 360.0      # 视场角
  filter_size_surf: 0.3  # 降采样体素大小 (m)
  filter_size_map: 0.3   # 地图降采样大小 (m)
```

**输出**:
- `/Odometry`: 里程计位姿 + 速度
- `odom→base_link` TF: 高频位姿变换
- `/cloud_registered`: 配准后的点云 (odom坐标系)
- `/path`: 机器人轨迹

**倒装注意事项**:
- MID-360 倒装后, 内置 IMU 的 z 轴指向地面
- FAST-LIO2 初始化时需要感知重力方向, 若 IMU z 轴反向可能导致初始化失败
- 解决方案: 在 extrinsic_R 中体现翻转, 或用 IMU 翻转节点预处理

### 5.3 robocon_perception — 障碍物检测

**处理流水线**:

```
/cloud_registered (PointCloud2, odom系, ~10Hz)
  │
  ├─ ① PassThrough ROI 过滤
  │    x: [-5.0, 5.0]  y: [-5.0, 5.0]  z: [-1.0, 1.5]
  │
  ├─ ② VoxelGrid 降采样
  │    leaf_size: 0.05m (减少计算量 ~10×)
  │
  ├─ ③ StatisticalOutlierRemoval 统计滤波
  │    meanK: 50, stddev: 1.0
  │
  ├─ ④ RANSAC 地面分割
  │    模型: PLANE, 距离阈值: 0.03m, 最大迭代: 200
  │    ├── 地面点 → ⑤ 强度方差分析 (检测砂砾/木屑, >500判为砂砾坑)
  │    └── 非地面点 → ⑥ Euclidean 聚类
  │         tolerance: 0.08m, min: 15, max: 20000
  │         │
  │         └── ⑦ 逐聚类特征提取 + 分类
  │              ├── PCA 特征值分析 (线性度/平面度/球度)
  │              ├── 3D 包围盒 (长宽高)
  │              ├── 离地高度 (相对于RANSAC平面)
  │              └── 分类决策树 ↓
  └── ⑧ 时间稳定化 (连续5帧确认)
       └── 发布 /obstacle_info + /obstacle_markers
```

**分类决策树** (按优先级):

| 障碍物 | 判定条件 | 置信度 |
|--------|---------|--------|
| 绕杆 (POLE) | 线性度>0.8, 高≥0.5m, 宽/深≤0.08m | 0.85 |
| 限高杆 (HEIGHT_BAR) | 离地0.25-0.35m, 长≥0.8m, 细, 线性度>0.7 | 0.75 |
| 高墙 (HIGH_WALL) | 平面度>0.7, 高≥0.3m, 宽≥0.5m | 0.80 |
| T台阶 (T_STEPS) | 高>0.16m, 长>0.4m, 非单平面(平面度<0.5) | 0.65 |
| 斜坡 (SLOPE) | 平面度>0.6, 法向量偏离重力5°-45°, 长宽>1m | 0.70 |
| 木桥 (BRIDGE) | 平面度>0.7, 离地>0.1m, 长宽>0.5m | 0.65 |
| 砂砾坑 (GRAVEL_PIT) | 地面点强度方差>500 | 0.60 |

**关键调参项**:

| 参数 | 默认值 | 调参建议 |
|------|--------|---------|
| `voxel_leaf_size` | 0.05 | 增大减少计算量(0.1), 减小提高精度(0.03) |
| `ground_distance_threshold` | 0.03 | 地面粗糙时增大(0.05), 平滑地面可减小 |
| `cluster_tolerance` | 0.08 | 障碍物密集时减小(0.05), 稀疏时增大(0.12) |
| `min_cluster_size` | 15 | 远距离障碍物可减小(10), 减少噪点误检可增大(25) |
| `stabilization_frames` | 5 | 增大减少误检(8), 减小提高响应速度(3) |

### 5.4 robocon_apriltag — 视觉标签

**架构**: Python 检测 + C++ 位姿修正

#### Python 检测节点

```python
# pupil_apriltags 库参数
Detector(
    families='tag36h11',       # 36位 hamming距离11
    nthreads=2,                # Orin Nano 用2线程
    quad_decimate=1.0,         # 不降采样
    quad_sigma=0.8,            # 增加模糊处理噪点
    refine_edges=1,            # 边缘细化
    decode_sharpening=0.25,    # 解码锐化
)
```

- 标签尺寸: 0.16m (16cm)
- 检测帧率: ~20-30fps @640×480
- 延迟: <10ms

#### C++ 位姿修正节点

**核心算法**:
```
1. 接收 /tag_detections (Python节点发布)
2. 查找最高置信度的已知Tag
3. 计算: T_map_base = T_map_tag × inv(T_camera_tag) × T_camera_base
4. 异常值拒绝:
   - 平移跳变 > 0.5m → 拒绝
   - 旋转跳变 > 15° → 拒绝
5. EMA 滤波:
   - 平移: EMA(α=0.3)
   - 旋转: 四元数 SLERP(α=0.3)
6. 发布 /tag_pose_correction (含协方差)
7. 最小修正间隔: 0.5s
```

**Tag 布局建议** (13个标签):

| ID | 位置 | 用途 |
|----|------|------|
| 0 | 起点区 | 初始定位验证 |
| 1-2 | 绕杆区入口/出口 | 确认进入/离开绕杆区 |
| 3 | 砂砾坑前 | 接近触发 |
| 4 | 限高杆前 | 接近触发 |
| 5-6 | 大斜坡起止 | 坡道起止确认 |
| 7 | 木桥A前 | 接近触发 |

| 11 | 高墙前 | 接近触发 |
| 12 | 终点/起点 | 返回确认 |

### 5.5 robocon_state_machine — 任务状态机

**顶层状态**:

```
INIT → WAITING_FOR_START → NAV_TO_OB → APPROACH_OB → CROSS_OB → ... → NAV_TO_END → COMPLETE
                                                    ↑ 循环8次                    ↓
                                                    └──────── RECOVERY ←─────────┘
```

**状态说明**:

| 状态 | 功能 | 控制方式 |
|------|------|---------|
| INIT | 等待定位就绪, TF树检查 | 无输出 |
| WAITING_FOR_START | 等待外部启动信号 | STOP |
| NAV_TO_OBSTACLE | Nav2导航至障碍物前方2m | /cmd_vel (Nav2) |
| APPROACH_OBSTACLE | 精细接近, 点云距离反馈 | /robot_command (直接) |
| CROSS_OBSTACLE | 障碍物特异性动作序列 | /robot_command (直接) |
| NAV_TO_END | 返回终点区 | /cmd_vel (Nav2) |
| MISSION_COMPLETE | 任务完成 | STOP |
| RECOVERY | 错误恢复 | STOP → 等待 → 重试 |

**障碍物子状态机** (以限高杆为例):

```
BAR_NAV ──→ BAR_DETECT ──→ BAR_APPROACH ──→ BAR_CROUCH ──→ BAR_CRAWL ──→ BAR_CLEAR ──→ BAR_STAND
 (Nav2)     (确认检测)      (精细接近)       (下蹲指令)     (匍匐前进)     (确认通过)     (站起)
```

**各障碍物处理器**:

| 处理器 | 子状态数 | 关键逻辑 |
|--------|---------|---------|
| PoleWeaveHandler | 3 | S形绕杆, Tag确认入口/出口 |
| GravelPitHandler | 4 | 砾石步态→普通步态切换, 强度检测 |
| HeightBarHandler | 4 | 下蹲→匍匐→站起, 限高300mm |
| SlopeHandler | 5 | 爬坡→坡顶行走≥1m→下坡, IMU俯仰检测 |
| BridgeHandler | 3 | 慢速过桥 0.08m/s |
| TStepsHandler | 5 | 台阶步态上下, 每阶需足部接触 |
| HighWallHandler | 4 | 垂直对齐→攀爬→落地恢复 |

### 5.6 robocon_localization — 定位管理器

**核心功能**: 发布 `map→odom` TF 变换

**工作模式**:
1. **启动默认**: 发布 Identity (map与odom重合)
2. **ICP 初始对齐**: 接收 `/initialpose`, 一次性设置
3. **AprilTag 修正**: 接收 `/tag_pose_correction`, EMA平滑更新

**参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `ema_alpha` | 0.3 | EMA平滑因子, 越小越平滑但响应慢 |
| `max_translation_jump` | 0.5m | 平移跳变阈值, 超过则拒绝修正 |
| `max_yaw_jump` | 15° | 旋转跳变阈值, 超过则拒绝修正 |
| `tf_rate` | 100Hz | TF发布频率 |

### 5.7 robocon_command_bridge — 控制桥接

**功能**: Nav2导航模式 ↔ 直接控制模式 仲裁切换

```
NAVIGATION模式 (默认):
  /cmd_vel (Nav2) → 透传 → 控制板

DIRECT模式 (障碍物跨越时):
  /robot_command (状态机) → 转发 → 控制板
  /cmd_vel 被抑制

模式切换:
  - 收到 /robot_command → 切换到 DIRECT
  - 2秒内无新的 /robot_command → 自动恢复 NAVIGATION
```

## 6. 定位系统

### 6.1 总体定位策略

```
三步定位:
  ① FAST-LIO2 提供高频局部里程计 (odom→base_link, 100Hz)
  ② localization_manager 提供全局定位 (map→odom, 100Hz)
  ③ AprilTag 提供间歇性全局修正 (0.5-2Hz, 取决于视野)

漂移控制:
  - FAST-LIO2 漂移率: 路程的 1-3% (14m场地 ≈ 14-42cm)
  - Tag 修正间隔: 每1-2个障碍物放置1个Tag
  - 漂移累积: 在两次Tag修正间累积, 修正时平滑跳变
```

### 6.2 ICP 初始全局定位

```bash
# 1. 建图阶段 (赛前)
遥控机器人走遍全场 → FAST-LIO2 累积 PCD → ros2 service call /map_save

# 2. 定位阶段 (比赛)
机器人放置启动区 → ICP 将当前 /cloud_registered 与 PCD 对齐 → 设置 map→odom
```

**ICP 参数** (icp.yaml):

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `rough_leaf_size` | 0.4 | 粗配准降采样 (m) |
| `refine_leaf_size` | 0.1 | 精配准降采样 (m) |
| `rough_iter` | 10 | 粗配准迭代次数 |
| `refine_iter` | 5 | 精配准迭代次数 |
| `thresh` | 0.15 | ICP 收敛阈值 (m) |
| `xy_offset` | 0.2 | 多假设搜索的xy步长 (m) |
| `yaw_offset` | 30° | 多假设搜索的偏航步长 |
| `yaw_resolution` | 10° | 偏航搜索分辨率 |

### 6.3 AprilTag 修正精度

| 因素 | 影响 | 优化 |
|------|------|------|
| Tag 距离 | <2m 时精度最佳 (±2cm), >5m 精度下降 | 在关键位置放置Tag, 距离<3m |
| 相机分辨率 | 640×480 足够, 更高分辨率提高远距离精度 | 用 D455 的 1280×720 |
| Tag 尺寸 | 16cm 在 <3m 可检测 | 远距离可用更大尺寸Tag |
| 光照 | 室内光照稳定, 无影响 | 避免Tag反光/阴影遮挡 |
| 移动模糊 | 高速移动时检测率下降 | 检测时减速或短暂停止 |

### 8.1 Nav2 配置

#### 全局代价地图 (global_costmap)

| 参数 | 值 | 说明 |
|------|-----|------|
| `global_frame` | map | 全局坐标系 |
| `width × height` | 30×30m | 覆盖14×6m场地 |

**static_layer**: 如有预建2D地图则加载为基础层, 无则留空
**obstacle_layer**: 实时 `/scan` 数据标记障碍物 (10m范围)

| 参数 | 值 | 说明 |

| `voxel_layer` | 标记+清除 | 3D体素层 |
| `inflation_radius` | 0.2m | 膨胀半径 |

#### DWB 控制器参数

| 参数 | 值 | 调参说明 |
|------|-----|---------|
| `max_vel_x` | 0.26 m/s | 四足步行最大速度, 过快易失稳 |
| `max_vel_theta` | 0.8 rad/s | 转向速度, 四足不能原地旋转 |
| `min_vel_x` | -0.1 m/s | 允许微退 |
| `acc_lim_x` | 1.0 m/s² | 加速度限制 |
| `acc_lim_theta` | 1.5 rad/s² | 角加速度限制 |

| `controller_frequency` | 20 Hz | 控制频率 |

#### 规划器

- 类型: NavfnPlanner (Dijkstra, A*关闭)
- 容差: 0.5m
- 允许通过未知区域

## 9. 状态机与控制

### 9.1 命令协议

#### /cmd_vel (Nav2 正常导航)

```
geometry_msgs/Twist:
  linear.x:  前进速度 (m/s),  范围 [-0.1, 0.3]
  angular.z: 转向速度 (rad/s), 范围 [-0.8, 0.8]
```

#### /robot_command (障碍物跨越)

```
RobotCommand:
  STOP:        紧急停止
  WALK:        行走, value=速度(m/s)
  TURN_LEFT:   左转, value=角速度(rad/s)
  TURN_RIGHT:  右转
  CROUCH:      下蹲, value=目标高度(m)
  STAND:       站起
  CLIMB:       攀爬, value=攀爬模式
  SPECIAL_GAIT: 特殊步态, gait_type=砂砾/匍匐/台阶
```

### 9.2 检查点持久化

```yaml
# checkpoint/mission_state.yaml
current_obstacle: 3     # 当前障碍物编号 (0-based)
state: 3                # MissionState枚举值
```

- 每次障碍物完成后自动保存
- 启动时加载, 支持从中断点恢复
- 手动删除 checkpoint 文件可从头开始

### 9.3 超时与错误处理

| 条件 | 动作 |
|------|------|
| 任务总时间 > 210s | 跳过当前 → 导航至终点 |
| 导航目标失败 (30s) | RECOVERY → 等待3s → 重试 |
| 障碍物跨越超时 (30s) | 后退0.3m → 重试 |
| 定位丢失 (2s无TF) | STOP → 等待Tag重定位 |
| 预期Tag 5s未检测到 | 用里程计死推算继续 |
| AprilTag修正跳变 > 0.5m | 拒绝修正, 记录警告 |

## 12. 参数调优指南

### 12.1 FAST-LIO2 里程计质量

| 问题 | 可能原因 | 调参方案 |
|------|---------|---------|
| 点云飘移/跳动 | 降采样太激进 | `filter_size_surf` 从 0.3 → 0.2 |
| 初始化失败 | IMU零偏大 | 静止3秒以上再启动; 增大 `acc_cov` |
| z轴漂移 | 地面约束不足 | 缩小 `det_range` 到 40m |
| 计算慢 | Orin Nano 限性能 | `point_filter_num` 从 3 → 5; 开启 OpenMP |

### 12.2 Nav2 导航性能

| 问题 | 调参方案 |
|------|---------|
| 绕障碍物太近 | 增大 `inflation_radius` (局部0.2→0.3, 全局0.25→0.35) |
| 无法到达目标 | 增大 `xy_goal_tolerance` (0.2→0.3m), 增大 `yaw_goal_tolerance` |
| 路径振荡 | 增大 `Oscillation` critic 权重; 降低 `max_vel_theta` |
| 导航太慢 | 增大 `max_vel_x` (0.26→0.35), 降低 `controller_frequency` |
| 在狭窄通道卡住 | 减小 `robot_radius` (0.2→0.15, 如实际尺寸更小) |

### 12.3 障碍物检测

| 问题 | 调参方案 |
|------|---------|
| 误检太多 | 增大 `stabilization_frames` (5→8); 增大 `min_cluster_size` |
| 漏检绕杆 | 减小 `pole_min_height` (0.5→0.3), 减小 `cluster_tolerance` |
| 限高杆检测不到 | 调整 `bar_z_min/max` 范围 (±0.02m) |
| 砂砾坑误判 | 调整强度方差阈值 (500→根据实际数据标定) |
| ROI 范围不够 | 根据实际场地调整 `roi_min/max_x/y/z` |

### 12.4 AprilTag 定位

| 问题 | 调参方案 |
|------|---------|
| 定位跳变太大 | 减小 `ema_alpha` (0.3→0.15); 减小 `max_translation_jump` |
| 修正太慢 | 增大 `ema_alpha` (0.3→0.5); 减小 `min_correction_interval` |
| Tag检测不到 | 检查光照; 增大 `quad_sigma` (0.8→1.2); 用更大Tag |
| 假阳性检测 | 增大 `decode_sharpening` (0.25→0.5); 增大最小置信度 |

### 12.5 整体时间预算 (210秒)

| 阶段 | 预估时间 | 优化空间 |
|------|---------|---------|
| 初始化 (TF就绪) | ~3s | 并行启动 |
| 8个障碍物间导航 | ~80s (10s/段) | 提高 max_vel_x |
| 8个障碍物跨越 | ~80s (10s/个) | 优化步态时长 |
| 返回起点 | ~15s | |
| **总计** | **~178s** | 余量 32s |

## 15. 常见问题排查

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| TF树只有base_link之后 | LiDAR驱动未启动 | 检查 livox_ros_driver2 是否编译; 检查 /livox/lidar 话题 |
| "传感器超出地图边界" | static_layer无地图 | 加载2D地图或不加载(自动降级) |
| FAST-LIO2初始化失败 | IMU倒装重力反向 | 在 extrinsic_R 中翻转z轴 |
| Nav2不发 /cmd_vel | AMCL未激活或无初始位姿 | rviz中发送 "2D Pose Estimate" |
| 障碍物检测为空 | 点云话题配置错误 | 检查 /cloud_registered 话题名和frame_id |
| AprilTag不检测 | pupil_apriltags未安装 | `pip3 install pupil-apriltags` |
| 编译 pcd2pgm 失败 | rclcpp_components缺失 | `sudo apt install ros-humble-rclcpp-components` |
| tmux 无法连接 | 旧session残留 | `tmux kill-session -t robo2026` |
| 路径拼接错误 | pcd2pgm未加斜杠 | 已修复, 拉取最新代码 |
