# 导航指南

## 概述

有了 2D 地图后，`master.launch.py` 已经集成了完整的导航栈：

```
FAST-LIO2(里程计) + pointcloud_to_laserscan(2D扫描)
+ localization_manager(map→odom TF)
+ Nav2(AMCL定位 + 全局规划 + DWB局部控制)
+ AprilTag(位姿修正)
```

只需启动时传入地图路径即可。

---

## 1. 启动导航

```bash
source ~/2026Obstacle-race/install/setup.bash

# 带 RViz
ros2 launch robocon_bringup master.launch.py \
    use_rviz:=true \
    map:=/home/dandelion/fast-lio-nav-obstacle/maps/field_map.yaml
```

`map` 参数默认值为空（不加载静态地图），传入 YAML 路径后 Nav2 会加载对应 PGM。

> 如果还没生成 2D 地图，参考 [建图指南.md](建图指南.md) 第 5 节。

---

## 2. 初始定位

机器人启动时 `map` 和 `odom` 在原点重合。需要用以下方式之一告诉它真实位置：

### 方式 A：AprilTag 自动修正（推荐）

机器人行驶中看到 AprilTag 后，`tag_pose_corrector` 自动计算全局位姿并修正 `map→odom` 变换。无需手动操作。

### 方式 B：RViz 手动设置

1. RViz 工具栏点击 **"2D Pose Estimate"**
2. 在地图上机器人实际位置单击并拖拽出朝向
3. AMCL 粒子云会聚拢到该位置

### 方式 C：ICP 自动对齐

`localization_manager` 启动时会用预建 PCD 点云与当前扫描做 ICP 配准：

```bash
ros2 launch robocon_bringup master.launch.py \
    map:=/path/to/map.yaml \
    pcd_map_path:=/home/dandelion/fast-lio-nav-obstacle/maps/field_map.pcd
```

---

## 3. 发送导航目标

### RViz 方式

RViz 工具栏点击 **"Nav2 Goal"**，在地图上单击目标位置并拖拽出朝向 → Nav2 自动规划并执行。

### 命令行方式

```bash
ros2 topic pub /goal_pose geometry_msgs/msg/PoseStamped \
  "{header: {frame_id: 'map'}, pose: {position: {x: 2.0, y: 1.0, z: 0.0}, orientation: {w: 1.0}}}"
```

## 4. 导航栈节点说明

| 节点 | 输入 | 输出 | 作用 |
|------|------|------|------|
| `fastlio_mapping` | `/livox/lidar` + `/livox/imu` | `/Odometry`, `odom→base_link` TF | 激光里程计 |
| `pointcloud_to_laserscan` | `/cloud_registered` | `/scan` | 3D→2D，供 AMCL 匹配地图 |
| `localization_manager` | `/Odometry`, `/tag_pose_correction` | `map→odom` TF (100Hz) | 全局定位融合 |
| `apriltag_detector` | `/camera/color/image_raw` | `/tag_detections` | 标签检测 |
| `tag_pose_corrector` | `/tag_detections` | `/tag_pose_correction` | 标签→全局位姿修正 |
| Nav2 `AMCL` | `/scan`, `map→odom`, 2D地图 | `map→base_link` TF | 粒子滤波全局定位 |
| Nav2 `Planner` | 代价地图, 目标点 | 全局路径 | Dijkstra 全局规划 |
| Nav2 `DWB` | 全局路径, 局部代价地图 | `/cmd_vel` | 局部轨迹控制 |

---

## 5. TF 树

```
map ──→ odom ──→ base_link ──→ livox_frame (LiDAR, 倒装)
                              ├─→ camera_link (D455)
                              └─→ imu_link (与LiDAR共体)
```

- `map→odom`：`localization_manager` 发布，由 AprilTag 修正
- `odom→base_link`：FAST-LIO2 发布，高频里程计
- 静态 TF：`base_link→livox_frame`, `base_link→camera_link`, `base_link→imu_link`

## 6. RViz 操作速查

| 操作 | 工具 | 快捷键 |
|------|------|--------|
| 设置初始位姿 | 2D Pose Estimate | 点击地图 |

## 7. 关键配置

### Nav2 参数

`src/navigation/robot_navigation2/config/nav2_params.yaml`

| 参数 | 值 | 说明 |
|------|-----|------|
| AMCL `min_particles` / `max_particles` | 500 / 2000 | 粒子数 |
| AMCL `laser_model_type` | `likelihood_field` | 似然场模型 |
| AMCL `global_frame_id` | `map` | 全局坐标系 |
| DWB `max_vel_x` | 0.26 m/s | 最大线速度 |
| DWB `max_vel_theta` | 0.8 rad/s | 最大角速度 |
| DWB `xy_goal_tolerance` | 0.25 m | 目标容忍半径 |

### 3D→2D 扫描参数

`master.launch.py` 中 `pointcloud_to_laserscan` 节点参数：

| 参数 | 值 | 说明 |
|------|-----|------|
| `min_height` | -0.3 m | 地面以下 |
| `max_height` | 0.8 m | 障碍物高度 |
| `angle_increment` | 0.0087 rad | ~0.5° 分辨率 |
| `range_max` | 30.0 m | 最大距离 |
